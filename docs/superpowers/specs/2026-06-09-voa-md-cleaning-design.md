# VOA .md 文本清洗脚本设计

## 概述

编写脚本批量处理 `VOA/` 目录下的 `.md` 文件，去掉标题、子标题、词汇表等无用内容，将段落按句子拆分（每句一行），输出为同名 `.txt` 文件。

## 脚本信息

- **文件路径**：`packages/server/src/tasks/cleanVoaMd.ts`
- **运行方式**：`pnpm --filter server exec tsx src/tasks/cleanVoaMd.ts`
- **输入**：`VOA/<article>/*.md`
- **输出**：`VOA/<article>/*.txt`（每句一行，句间空行分隔）

## 处理流程

### 阶段一：行级预处理

对 .md 文件的每一行进行判断，确定保留/删除：

1. **删除标题行**：第一个非空行如果以 `#` 开头 → 删除
2. **删除 `No media source` 行**：第二个非空行如果结尾无标点（非 `.!?:`）→ 删除
3. **删除 `---` 及之后所有行**：词汇表区域
4. **删除子标题行**：满足「独立行 + 结尾非 `.!?:`」→ 删除
5. **删除问句式短标题**：满足「独立行 + `?` 结尾 + 长度 ≤ 80 字符」→ 删除

「独立行」定义为：该行前后均为空行（或文件首/尾边界）。

### 阶段二：句子拆分

对保留的正文行进行句子级拆分：

1. **缩写保护**：用占位符替换已知缩写中的 `.`
   - 白名单：`U.S.` `Dr.` `Mr.` `Mrs.` `Jr.` `Sr.` `St.` `a.m.` `p.m.` `e.g.` `i.e.`
   - 模式兜底：`. ` 后跟小写字母 → 非句号
2. **数字小数点保护**：`数字.数字` 模式中的 `.` 不视为句号
3. **省略号保护**：`. . .` 或 `...` 不视为句号
4. **引号保护**：用栈匹配 `"..."` 配对，引号内文本不参与拆分
5. **按 `[.!?]` 拆分**：对非保护区域按句子结束符切割
6. **还原占位符**：恢复缩写、数字中的 `.`
7. **引号保留**：引号本身不删除

### 阶段三：输出

- 每句一行
- 句子之间空行分隔
- 写入与原 .md 同名的 `.txt` 文件

## 边界覆盖

| 边界                                    | 处理方式                       |
| --------------------------------------- | ------------------------------ |
| 缩写 `U.S.`, `Dr.`, `Jr.` 等            | 白名单占位保护 + 模式推断      |
| 数字小数点 `$6.2`, `1.7 kilograms`      | 数字`.`数字不视为句号          |
| 省略号 `. . .` / `...`                  | 白名单保护                     |
| 引号内 `.` `!` `?`                      | 引号栈保护，内部不拆分         |
| `?` 结尾问句式标题（短）                | 长度 ≤ 80 → 删除               |
| `?` 结尾正文疑问句（长）                | 长度 > 80 → 保留并正常拆分     |
| `:` 结尾行（引述引导语）                | `:` 属于有效结尾，不视为子标题 |
| 空行                                    | 忽略，不输出                   |
| 名词短标题 `Difficulties with hydrogen` | 无标点结尾 → 删除              |
| 引述引导语 + 引号内句子                 | 整句保留，引号内部不拆分       |

## 关键决策

- **子标题识别**：无标点结尾 OR（`?` 结尾 AND 短长度）→ 子标题
- **缩写处理**：白名单 + 模式推断双保险
- **引号处理**：保留引号，引号内不拆分
- **输出方式**：同名 .txt，不覆盖原 .md
- **`:` 结尾行**：视为正文，保留
